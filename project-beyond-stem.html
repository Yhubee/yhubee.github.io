Beyond STEM – Learning Tool (Full-stack MVP)

Re-engaging students in non-STEM subjects with a game-based, full-stack learning experience.

Role: Full-stack developer & UX collaborator

Context: Week 5 Project Alpha at La Fosse

Domains: Non-STEM education, game-based learning, engagement design

Methods: Agile teamwork, UX design, full-stack implementation, SQL schema design, automated testing

← Back to Projects & Case Studies

Overview

Beyond STEM is a team-built educational tool designed to re-engage students in non-STEM subjects across a network of secondary schools. It was built as a full-stack MVP that combines UX design, Agile collaboration, backend and frontend development, and SQL database integration.

Challenge

The Hive group of schools reported decreased student interest and retention in non-STEM subjects. The mission was to create a fun digital solution that puts student enjoyment at the centre of the learning experience.

Stakeholder Insights

  • Students are overwhelmed by repetitive lessons and struggle to retain information.
  • Teachers report reduced lesson frequency and more superficial coverage.
  • Parents see a drop in engagement and enjoyment.
  • Leaders want to support holistic education and critical thinking.

Our Solution

An interactive, game-based web app that delivers non-STEM knowledge in bite-sized historical fact cards with images. Some facts lean into the weird and wonderful to tap into teens’ curiosity and love of the unexpected.

Key Features

  • User authentication (signup / login).
  • Fact-based learning game mechanics.
  • Persistent data via a relational database.
  • Responsive frontend and deployed full-stack app.

Backlog Items

  • Multiple-choice quizzes and score tracking.
  • Feedback and richer progress analytics.
  • Admin / review access for teachers and parents.
  • Extending to other non-STEM subjects.

Technical Architecture

  • Frontend: HTML, CSS, JavaScript.
  • Backend: Node.js, Express.
  • Database: SQL.
  • Auth: Basic auth with hashed passwords.
  • Testing: Jest and Supertest (target 60%+, actual 78%).
  • Deployment: Render / Netlify.
  • Version control: Git + GitHub with feature-branch workflow.

Installation & Local Run

  1. Clone: git clone git@github.com:rafsanzi-ludhi/beyond_stem.git.
  2. cd beyond_stem.
  3. npm install.
  4. cd server and create .env with PORT=3000 and DB_URL.
  5. Run npm run dev (backend on http://localhost:3000).

API Endpoints

  • GET '/' – entry endpoint.
  • GET '/history' – get all history facts.
  • GET '/history/:id' – get one fact.
  • POST '/history' – create a new fact.
  • PATCH '/history/:id' – update a fact.
  • DELETE '/history/:id' – delete a fact.

Testing

Automated tests written with Jest and Supertest. Target coverage was 60%+, and the project achieved approximately 78% coverage.
