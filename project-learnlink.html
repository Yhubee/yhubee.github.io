LearnLink – Peer-to-Peer Skill Exchange Platform

Building a cash-free, community-driven skill-swapping MVP with end-to-end full-stack delivery.

Role: Full-stack developer & product collaborator

Context: Course full-stack MVP at La Fosse

Domains: Peer-to-peer learning, community platforms, data-driven dashboards

Methods: Agile teamwork, UX design, full-stack implementation, data modelling, technical documentation

← Back to Projects & Case Studies

Overview

LearnLink is a peer-to-peer educational platform that connects people who want to teach and learn skills in their local area. Users can offer skills like guitar, Python, or photography, find nearby skill-swappers, track progress, and build a learning network.

It was built as a full-stack MVP, integrating course topics such as UX design, Agile collaboration, backend and frontend development, database design, and data visualisation.

Challenge

Many people want to learn new skills but face barriers such as cost, access, and time. The goal was to create a collaborative, local, and cash-free skill-sharing tool that makes learning accessible, social, and efficient.

Stakeholder Insights

  • Learners feel limited by traditional platforms because of cost or rigid formats.
  • Skill-holders want to teach but lack a structured way to offer their time.
  • Communities want more local, inclusive, and informal learning experiences.

Our Solution

A location-based web app that matches users who want to learn skills with others willing to teach them. Users create profiles, post skills they can offer or want to learn, and connect for in-person or online swaps. No money, only learning.

Key Features

  • User authentication (signup / login).
  • Location-based skill matchmaking.
  • Persistent data via a relational database.
  • Skill offer / request posting.
  • Time-availability calendar.
  • In-app messaging and request handling.
  • Responsive frontend UI.

Backlog Items

  • Profile ratings and reviews.
  • Swap history and analytics.
  • Mobile app version.
  • Social media profile sharing.
  • Swap request flow before chat initiation.
  • QR code-based session logging.
  • General location display (Google Maps API).
  • Visualisations like total learning time per skill and learning streaks.

Data Architecture

LearnLink uses a star schema with dimension and fact tables:

  • dim_user: user profiles.
  • dim_skill: skill categories and descriptions.
  • dim_time: time breakdowns for learning sessions.
  • facts_learning: skills a user is learning.
  • facts_teaching: skills a user is teaching.
  • facts_session: logged peer-to-peer sessions.

PostgreSQL stores the structured data that powers dashboard visualisations.

Technical Architecture

  • Frontend: HTML, CSS, JavaScript.
  • Backend: Node.js, Express, Socket.io.
  • Database: SQL (PostgreSQL).
  • Auth: Basic authentication with hashed passwords.
  • Testing: Jest and Supertest.
  • Deployment: Render / Netlify.
  • Version control: Git + GitHub using feature-branch workflow.

Installation & Local Run

  1. Clone: git clone git@github.com:bbm2910/LearnLink.git.
  2. cd LearnLink.
  3. npm install.
  4. cd server and create a .env file with:
    • PORT=3000
    • DB_URL='Your Database URL'
  5. Start backend: npm run dev (runs on http://localhost:3000).

Using the App

  • Sign up or log in.
  • Add skills you can teach and skills you want to learn.
  • Search or browse nearby skill-swappers.
  • Connect and arrange your first skill swap.
